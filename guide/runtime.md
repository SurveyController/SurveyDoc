# 运行参数

这是一个很有价值的设置页面，十分建议仔细阅读和配置，几乎每一个选项都直接关系到你问卷的提交质量。

由于版本更迭在 UI 上的差异，以下截图可能和你看到的界面不完全一样，但大体逻辑都是没有变化的，看着配就行。

![作答设置](../assets/runtime_1.png)

## 目标份数 & 并发数

这两个选项几乎是所有用户的必填配置项，在主页面也专门有这两个设置项的快捷入口。主页和运行参数页的这两个设置项是同步的，修改任一处都会同步更新另一个位置的显示。

至于**并发数**，简单来说就是提交速度。并发数越大，提交得越快，适用于问卷情况特别紧急的用户。

:::danger 🚫警告
过高的并发数可能会导致电脑极度卡顿，不要无脑地追求过高的并发！除非你真的很紧急
:::

下面以`无头模式`下提交 1000 份问卷数量为例，来说明不同并发数的提交效果：
| 并发数 | 建议的最低配置 | 大约所需时间 |
|-------|-------|-------|
| 5 | 4核8G | 1小时 |
| 16 | 8核16G | 40分钟 |
| 32 | 8核24G | 25分钟 |
| 64 | 12核24G | 15分钟 |

> 所以真的没必要拉高并发，又不是什么火烧眉毛的事情，慢工出细活没什么不好的。

:::tip 💡提示
Playwright 的同步并发并不是线程安全的，创建、销毁、资源占用开销都是特别高的

因此，本程序采用了 async 异步单实例多 context 并发来极大降低资源占用，大幅提高性能
:::

## 提升问卷信度

:::warning ⚠️注意
此设置项仅对`量表题`、`矩阵量表题`、`评价题`生效！
:::

标准的评估与调查问卷是采用严格的五级量表题型来设计的，采用诸如“`非常同意` `同意` `一般` `不同意` `非常不同意`”之类的选项内容。本程序对多级量表也提供支持。

> ⚠ 部分用户反映测试出来的信度过低，往往是他们采用了单选题而非量表题的题型设计，**属于问卷设计者自身的错误与失责**，非本程序问题！

支持自定义预期Cronbach's α系数，但最终以专业 SPSS 软件分析结果为准，此处仅供参考，不做保证。

## 无头模式

建议开启。启用则使浏览器在后台运行，不弹窗，资源占用小，不影响其他工作。

:::tip 💡提示
若作答过程存在异常问题，建议关闭无头模式提交1份尝试复现。
:::

## 提交间隔

即两次提交的问卷之间等待的时长。程序会在设定值附近做一点随机浮动。

绝大多数情况下，没必要调整此处的设置，保持默认为 0 即可，除非你真的有特殊需求。

![时间控制设置](../assets/time_1.png)

如果你只是先测问卷是否能跑通，或者根本就没哪个闲着蛋疼要求你展示后台的详细作答情况，这里直接填 0 没有任何问题，反正问卷又不会被封禁。

反正真到了火烧眉毛的时候，这些时间控制啥的也都别管了，全部填 0 赶快填完再说吧…

## 作答时长

单份问卷大概耗时多久。程序会在你设定值附近做一点随机浮动。

:::tip 💡提示
针对问卷星问卷，可通过修改 starttime 值可以实现伪造作答时长

即 5 秒钟完成提交，但后台显示 5 分钟甚至更长的时间
:::

## 定时模式

一旦开启，就会忽略上述两个时间设置，改成在开放后立刻高频刷新并尽快提交。

:::tip 💡提示
适用于`抢志愿名额` `抢选课名额`等类似场景，非特殊需求请勿开启
:::
